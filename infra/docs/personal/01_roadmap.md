# 🗺️ 개인 로드맵 (GitCat Project Roadmap)

이 문서는 프로젝트가 끝날 때까지 수행해야 할 **모든 핵심 작업과 전략적 의사결정의 이유(Why)**를 전체적으로 기록하고 관리하는 마스터 플랜입니다. 기능 구현 순서에 맞춰 페이즈(Phase)별로 구성되어 있습니다.

---

## 🎯 프로젝트 최종 목표
- **안전한 로컬 환경:** 사용자 코드 유출 없이 100% 로컬 파일/SQLite 기반으로 작동하는 확장 프로그램 개발.
- **스마트한 병합 경험:** AI가 단순 텍스트를 넘어 코드 구조(AST)를 이해하여 충돌을 예측하고 안전한 병합 초안을 제시.
- **신뢰성 보장:** 정적 분석(TypeScript Compiler, ESLint)을 통한 AI 할루시네이션 완벽 방어.

---

## 📅 페이즈 1: 코어 인프라 구축 (Infra) - [완료]
> **목표:** 모노레포 환경의 빌드 안정성과 로컬 스토리지 체계를 갖춰, 이후 모든 개발 시 데이터가 안전하게 저장/조회될 수 있는 토대 마련.

- [x] **모노레포 CI/CD 파이프라인 구축**
  - **Why:** 패키지 간 의존성 문제를 방지하고, 코드 병합 시 Lint/Build 자동 검증으로 품질 유지.
- [x] **로컬 파일 시스템(I/O) 유틸리티 세팅**
  - **Why:** 스냅샷 발생 시 즉각적으로 원본 파일을 평탄화하여 로컬에 안전하게 복사/원복하기 위함.
- [x] **SQLite 메타데이터 스키마 및 연동 (DAO 구조)**
  - **Why:** 스냅샷, 세션, AI 추론 결과 간의 복잡한 관계형 데이터를 빠르고 정확하게 쿼리하기 위함.

---

## 📅 페이즈 2: 확장 프로그램 뼈대 및 Git 코어 연동 (Scaffolding & Git)
> **목표:** VS Code 익스텐션으로서 구동할 수 있는 뼈대를 잡고, 사용자의 로컬 Git 상태를 제어할 수 있는 기반 모듈을 완성합니다.

- [ ] **VS Code Secrets 기반 API Key 관리 모듈 연동 (`packages/storage/src/secrets`)**
  - **Why:** AI 호출용 API 키를 평문으로 두지 않고 VS Code의 기본 보안 표준에 맞게 암호화하여 저장하기 위함.
- [ ] **`apps/extension` 메인 엔트리포인트 구성 및 기본 명령어 등록**
  - **Why:** 사용자가 Ctrl+Shift+P를 눌렀을 때 GitCat 명령어들이 정상적으로 호출되고 동작할 수 있도록 진입점을 뚫어주기 위함.
- [ ] **`packages/git-core` 모듈 초기화 및 스냅샷 자동화 로직 구현**
  - **Why:** AI 작업이 시작되기 직전, 현재 워크스페이스의 uncommitted 변경사항을 감지하고 자동으로 스냅샷(백업)을 찍는 핵심 Git 연동 로직 필요.

---

## 📅 페이즈 3: AI 파이프라인 및 충돌 분석기 개발 (AI Pipeline)
> **목표:** 기반 인프라 위에서 실제 비즈니스 로직(충돌 분석, 프롬프트 엔지니어링)을 결합하여 MVP의 핵심 가치를 증명합니다.

- [ ] **충돌 예측(Conflict Analyzer) 알고리즘 구현**
  - **Why:** 현재 브랜치와 타겟 브랜치 간의 단순 diff 비교를 넘어, 어떤 파일의 어느 영역이 충돌할지 사전에 예측하여 AI에게 컨텍스트로 제공하기 위함.
- [ ] **OpenAI API 연동 및 통신 레이어 구축**
  - **Why:** 외부 LLM 모델과 통신하며 토큰 사용량 추적, 재시도(Retry), 타임아웃 처리를 담당하는 안정적인 네트워크 모듈 구현.
- [ ] **AI 프롬프트 엔지니어링 및 응답 파싱(Prompt Utils)**
  - **Why:** LLM의 할루시네이션(환각)을 제어하고, 정형화된 JSON 형태로 병합 초안이나 추천 결과를 예측 가능하게 받아오기 위함.

---

## 📅 페이즈 4: UI 연동 및 정적 분석 검증 레이어 (UI & Validation)
> **목표:** 생성된 AI 결과물을 사용자가 확인하고 피드백할 수 있는 UI를 붙이고, AI의 응답이 코드 구조적으로 안전한지 문지기(Validator)를 세웁니다.

- [ ] **`apps/webview-ui` 연동 및 메시지 패싱(Webview ↔ Extension) 구현**
  - **Why:** 터미널 콘솔이 아닌, VS Code 내부의 깔끔한 React 기반 Webview 화면으로 AI의 제안을 띄워주고 사용자의 수락/거절 피드백을 받기 위함.
- [ ] **TypeScript Compiler API / ESLint 기반 정적 분석 검증기 도입**
  - **Why:** AI가 작성한 병합 초안 코드가 괄호를 빼먹거나 존재하지 않는 변수를 참조하는 등 문법적으로 유효하지 않은 코드인지 사용자 화면에 띄우기 '직전'에 1차 차단하기 위함 (Fail-safe Design).

---

## 📅 페이즈 5: 테스트, 최적화 및 릴리스 (Release)
> **목표:** 모든 기능이 통합된 상태에서 버그를 잡고 최종 MVP 버전을 패키징합니다.

- [ ] **종단간(E2E) 테스트 및 엣지 케이스 점검**
  - **Why:** 대용량 파일 병합 시나리오나, API Key가 없을 때의 동작 등 예외 상황에서 확장이 뻗지 않는지 검증.
- [ ] **익스텐션 패키징 (.vsix 생성) 및 로컬 배포**
  - **Why:** 실제 프로젝트 팀원들에게 로컬 확장 파일(`.vsix`)을 공유하여 실무에서 버그 없이 구동되는지 최종 알파 테스트 진행.
