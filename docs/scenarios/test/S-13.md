# S-13

기능: 사용자 반영
비고: 수정 후 반영 케이스
시나리오명: 사용자 수정 후 반영
유형: 정상

### 목적

사용자가 AI 제안을 그대로 수락하지 않고 일부 수정 후 반영할 수 있는지 확인한다.

### 사전 상태

- AI 병합 초안이 생성된 상태이다.

### 실행 절차

1. 사용자가 AI 제안 결과를 연다.
2. 제안 코드를 일부 수정한다.
3. 수정 후 반영 버튼을 선택한다.
4. GitCat이 수정된 결과를 파일에 적용한다.

### 기대 결과

- 사용자가 직접 수정한 내용이 최종 반영된다.
- AI 원안이 아닌 수정본 기준으로 저장된다.

### 성공 기준

- 최종 파일 상태가 사용자 수정본과 일치하면 성공
