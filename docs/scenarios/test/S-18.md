# S-18

기능: GUI
비고: GUI 기반 실행 여부, 결과 표시, 상태 반영
시나리오명: GUI로 Git 명령어 실행 확인
유형: 정상

### 목적

사용자가 GUI에서 Git 명령어를 직접 실행하고, 실행 결과가 화면에 정상 반영되는지 확인한다.

### 사전 상태

- Git 작업이 가능한 로컬 저장소가 준비되어 있다.
- GUI에서 실행 가능한 Git 작업 항목이 노출되어 있다.
- 실행 결과를 확인할 수 있는 화면 또는 메시지 영역이 존재한다.

### 실행 절차

1. 사용자가 Git 작업이 가능한 화면으로 이동한다.
2. GUI에서 실행 가능한 Git 명령어 또는 작업 항목을 확인한다.
3. 사용자가 특정 Git 작업을 선택한다.
4. 실행 버튼을 선택한다.
5. 실행 결과를 화면에서 확인한다.

### 기대 결과

- 사용자가 터미널 명령어 입력 없이 GUI에서 Git 작업을 실행할 수 있다.
- 실행 결과가 화면 또는 메시지로 표시된다.
- 실행 후 변경된 Git 상태가 UI에 반영된다.

### 성공 기준

- GUI에서 실행한 Git 작업의 결과가 정상 표시되고, 변경된 상태가 화면에 반영되면 성공
