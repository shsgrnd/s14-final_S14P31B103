# S-06

기능: 원복
비고: 원복 직전 상태가 새 스냅샷으로 추가되는지 검증
시나리오명: 원복 시 현재 상태 안전 스냅샷 적재 및 재복원
유형: 검토

### 목적

원복을 실행하기 전에 현재 상태를 안전 스냅샷으로 추가 저장하고, 이후 다시 재복원 가능한지 확인한다.

### 사전 상태

- 현재 작업 상태와 과거 스냅샷 상태가 다르다.

### 실행 절차

1. 사용자가 원복 대상 스냅샷을 선택한다.
2. GitCat이 원복 직전 현재 상태를 새 안전 스냅샷으로 저장한다.
3. 원복을 수행한다.
4. 새로 생성된 안전 스냅샷이 목록에 추가되었는지 확인한다.
5. 필요 시 방금 저장된 안전 스냅샷 기준으로 다시 복원해본다.

### 기대 결과

- 원복 직전 현재 상태가 별도 스냅샷으로 저장된다.
- 사용자는 이전 현재 상태로 다시 되돌릴 수 있다.

### 성공 기준

- 원복 전 상태가 새 스냅샷으로 추가되고 재복원 가능하면 성공
