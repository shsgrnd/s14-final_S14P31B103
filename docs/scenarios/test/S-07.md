# S-07

기능: 충돌 탐지
비고: 3-way diff + AST
시나리오명: 병합 전 충돌 후보 탐지 - 충돌 있음
유형: 정상

### 목적

실제 충돌 가능 구간이 있을 때 병합 전 후보 탐지가 정상 동작하는지 확인한다.

### 사전 상태

- 동일 파일 또는 동일 함수가 서로 다른 브랜치에서 수정되어 있다.

### 실행 절차

1. 사용자가 병합 전 분석을 실행한다.
2. GitCat이 3-way diff를 수행한다.
3. AST 기반 구조 비교를 수행한다.
4. 충돌 가능성이 높은 구간을 목록으로 표시한다.

### 기대 결과

- 충돌 후보가 탐지된다.
- 파일, 위치, 위험 구간이 함께 제공된다.

### 성공 기준

- 실제 충돌 가능 구간이 탐지 결과에 포함되면 성공
