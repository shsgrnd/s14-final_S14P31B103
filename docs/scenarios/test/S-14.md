# S-14

기능: 사용자 반영
비고: 미반영 상태 검증
시나리오명: Reject 및 병합 미완료
유형: Reject

### 목적

사용자가 AI 제안을 거절했을 때 병합이 완료되지 않고 미반영 상태가 유지되는지 확인한다.

### 사전 상태

- AI 병합 초안과 중재안이 생성된 상태이다.

### 실행 절차

1. 사용자가 결과 검토 화면을 연다.
2. Reject 버튼을 선택한다.
3. GitCat이 적용 없이 종료한다.
4. 결과 상태를 확인한다.

### 기대 결과

- 병합 결과가 파일에 반영되지 않는다.
- 미반영 또는 미완료 상태가 표시된다.

### 성공 기준

- Reject 후 실제 파일이 변경되지 않으면 성공
