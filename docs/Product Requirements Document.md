# 프로젝트 기획서

## 1. 프로젝트 명
AI 코드 작업 안전성 확보와 병합 전 충돌 중재를 지원하는 Git 확장 프로그램 개발

### 서비스명
SafeGit

## 2. 프로젝트 정의

### 2.1 한줄 정의
본 프로젝트는 AI를 활용한 코드 작업 과정에서 커밋되지 않은 로컬 작업 상태를 AI 작업 실행 단위로 자동 저장하고, 문제 발생 시 해당 작업 직전 상태로 원복을 지원하며, 병합 이전 단계에서 충돌 가능성을 예측하여 AI 기반 병합 초안 및 중재안을 제시하는 Git 확장 프로그램을 개발하는 것을 목표로 한다.

### 2.2 핵심 차별점
본 프로젝트의 핵심 차별점은 협업용 공식 이력인 커밋과 별도로, AI 작업 실행 단위의 경량 복구 체계를 제공한다는 점과, 병합 이후 충돌 해결이 아니라 병합 이전 단계에서 충돌 가능성을 예측하고 병합 초안 및 중재안을 제시한다는 점에 있다.

## 3. 해결하고자 하는 문제
본 프로젝트가 해결하고자 하는 문제는 다음과 같다.

- AI 기반 코드 수정 과정에서 예상하지 못한 변경이 함께 반영되어 작업 안정성이 저하될 수 있음
- 기존 커밋 이력은 협업용 공식 기록으로는 적합하지만, AI 작업 단위별 경량 복구 수단으로 사용하기에는 부담이 큼
- 커밋되지 않은 로컬 작업 상태에서 AI 작업 결과에 문제가 발생하였을 때, 사용자가 해당 작업 이전 상태로 신속하게 되돌리기 어려움
- 병합 과정에서 동일 코드 구간의 직접 충돌뿐 아니라, 인터페이스 변경, 공통 모듈 수정, 데이터 구조 변화 등으로 인한 간접 충돌이 발생할 수 있음
- 기존 Git 기반 병합 방식은 충돌 발생 이후의 사후 해결에 초점이 맞추어져 있어 선제적 대응이 어려움
- Git 숙련도가 낮은 사용자는 커밋 메시지 작성, 브랜치 관리, 불필요한 로컬 브랜치 정리 등 부가 작업에서도 부담을 느낄 수 있음

## 4. 프로젝트 개요
생성형 AI 기반 코딩 도구의 활용이 확대되면서, 개발자는 코드 생성, 수정, 리팩토링, 테스트 보완 등 다양한 작업을 AI와 함께 수행하고 있다. 그러나 AI가 코드에 직접 개입하는 작업은 짧은 시간 내 다수의 변경을 발생시키며, 사용자가 의도하지 않은 수정까지 함께 반영될 가능성이 있어 작업 안정성과 복구 가능성에 대한 요구가 커지고 있다.

본 프로젝트는 이러한 문제를 해결하기 위해 두 가지 핵심 기능을 제공한다. 첫째, AI 작업 수행 직전의 커밋되지 않은 로컬 작업 상태를 자동 스냅샷으로 임시 저장하고, 문제가 발생할 경우 해당 AI 작업 이전 상태로 원복할 수 있는 세이프티 레이어를 제공한다. 둘째, 병합 이전 단계에서 작업 중인 워크트리와 변경 내용을 분석하여 충돌 가능성을 사전에 식별하고, AI 기반 병합 초안 및 중재안을 제시한 뒤 사용자가 이를 검토하고 반영할 수 있도록 지원한다.

부가적으로 본 서비스는 커밋 메시지, 브랜치명, 작업 설명 추천 기능과 로컬 브랜치 정리 기능을 함께 제공하여 Git 사용 편의성과 작업 환경 정돈을 지원한다.

## 5. 개발 배경 및 필요성

### 5.1 개발 배경
최근 개발 환경에서는 생성형 AI를 활용하여 기능 구현, 리팩토링, 오류 수정, 테스트 코드 작성 등을 수행하는 사례가 급격히 증가하고 있다. 특히 반복 작업이나 초안 작성 과정에서 AI 활용도가 높아지면서 개발 생산성은 향상되고 있으나, 동시에 AI가 생성한 변경 사항을 사용자가 충분히 검토하지 못한 상태에서 반영하는 문제가 발생할 수 있다. 이로 인해 예상치 못한 코드 변형, 기능 오작동, 병합 실패, 복구 부담 증가 등의 문제가 나타나고 있다.

### 5.2 필요성
기존 Git 도구는 변경 이력 관리와 협업 기록에는 강점을 가지지만, AI 작업 단위별 보호와 복구를 위한 경량 안전 장치로 활용되기에는 한계가 있다. 커밋은 협업용 기록이자 의도적으로 남기는 공식 이력인 반면, AI 작업은 더 짧은 단위로 자주 발생하며 실험적 성격을 가지므로, 별도의 임시 복구 체계가 필요하다. 또한 병합 단계에서는 충돌 발생 이후 수동 해결에 의존하는 경우가 많아, AI 활용 환경에 적합한 사전 충돌 예측 및 중재 기능의 필요성이 높아지고 있다.

## 6. 타깃 사용자
본 프로젝트의 주요 사용자는 다음과 같다.

- AI 코딩 도구를 적극 활용하는 개인 개발자
- Git 숙련도가 높지 않지만 AI 기반 코드 작업을 수행하는 초급 개발자
- 다수의 AI 제안 또는 병렬 작업 결과를 병합해야 하는 팀 개발 환경의 구성원
- 커밋 이전 단계의 로컬 작업 보호와 병합 안정성을 동시에 확보하고자 하는 개발 조직

## 7. 용어 정의
본 프로젝트에서 사용하는 주요 용어는 다음과 같다.

- **세이프티 레이어**: AI 작업 전후 상태를 보호하고 원복을 지원하는 내부 안전 계층
- **자동 스냅샷**: AI 작업 실행 직전에 생성되는 임시 복구 기록으로, 커밋되지 않은 로컬 작업 상태를 저장함
- **하네스 프롬프팅**: AI 작업 안정성 강화를 위해 적용되는 표준 프롬프트
- **병합 초안**: 병합 후보 코드를 AI가 제안한 초안
- **중재안**: 충돌 원인 및 대안 방향을 사용자에게 설명하는 제안
- **정적 분석**: 문법 오류, 참조 불일치, 타입 충돌 등 구조적 문제를 사전에 점검하는 절차

## 8. 제안 서비스 개요
본 프로젝트에서 제안하는 서비스는 AI 기반 코드 작업을 보다 안전하게 수행할 수 있도록 지원하는 Git 확장 프로그램이다. 서비스는 크게 두 가지 핵심 축으로 구성된다.

첫째, AI 작업 전 상태를 자동으로 저장하는 세이프티 레이어를 제공한다. 해당 기능은 커밋처럼 협업용 공식 이력을 남기기 위한 목적이 아니라, AI 작업 중 문제가 발생하였을 때 빠르게 복구할 수 있도록 하는 경량 임시 저장 체계이다. 자동 스냅샷은 AI 작업 실행 직전에 생성되며, 커밋되지 않은 로컬 작업 상태를 기준으로 개별 AI 작업 실행 단위의 복구 지점을 제공한다. 이를 통해 사용자는 문제 발생 시 해당 AI 작업 수행 직전 상태로 원복할 수 있다.

둘째, 병합 이전 단계에서 작업 중인 워크트리와 변경 내용을 수집하여 충돌 가능성을 사전에 분석하고, AI 기반 병합 초안 및 중재안을 제공한다. 이후 정적 분석을 통해 제안 결과를 점검하고, 문법 오류, 참조 불일치, 타입 충돌 등 구조적 문제를 확인한 뒤 필요 시 보정안을 추가 제시함으로써 사용자가 최종 내용을 검토하고 병합할 수 있도록 지원한다.

부가적으로 본 서비스는 커밋 메시지, 브랜치명, 작업 설명 자동 추천 기능과 사용하지 않는 로컬 브랜치를 정리하는 기능을 제공하여 Git 사용 편의성과 작업 환경 정돈을 지원한다.

## 9. 핵심 기능

### 9.1 세이프티 레이어 기반 자동 스냅샷 및 원복 지원
본 프로젝트의 첫 번째 핵심 기능은 AI 작업 이전 상태를 자동으로 보존하는 세이프티 레이어이다. 자동 스냅샷은 협업용 공식 이력인 커밋을 대체하는 기능이 아니라, AI 작업 중 발생할 수 있는 예기치 않은 변경에 신속히 대응하기 위한 복구 중심의 경량 안전 장치라는 점에서 기존 Git 이력 관리 방식과 구별된다. 이를 통해 사용자는 AI 작업 중 문제가 발생한 경우 AI 작업 실행 단위로 이전 상태에 원복할 수 있다.

또한 AI 작업 수행 시 보다 일관되고 안전한 결과를 유도하기 위해, AI 작업 안정성 강화를 위한 표준 프롬프트(이하 하네스 프롬프팅)를 자동 적용하여 작업 실행 이전의 안전 장치를 강화한다.

#### 세부 기능
- AI 작업 실행 직전 자동 스냅샷 생성
- 스냅샷과 커밋 이력의 분리 운영
- AI 작업 실행 단위 원복 지원
- 하네스 프롬프팅 자동 적용
- 복구 시점 선택 및 반영 지원

#### 기능적 의의
- AI 작업 실수 또는 예기치 않은 코드 변형에 대한 즉각적 대응 가능
- 협업용 커밋 이력과 복구용 임시 기록의 역할 분리
- 커밋되지 않은 로컬 작업 상태 보호 가능
- 실험적 AI 작업에 대한 심리적·기술적 부담 완화
- 안전한 AI 코드 수정 환경 제공

### 9.2 병합 전 충돌 예측 및 AI 기반 병합 초안 제안
본 프로젝트의 두 번째 핵심 기능은 병합 이전 단계에서 작업 중인 워크트리와 변경 사항을 수집하여 충돌 가능성을 사전에 식별하고, AI 기반 병합 초안 및 중재안을 제시하는 기능이다. 본 기능은 동일 코드 구간의 직접 충돌뿐 아니라, 기능 간 연결 지점에서 발생할 수 있는 간접 충돌 가능성도 함께 고려한다.

이후 정적 분석을 통해 제안된 병합 초안의 문제 여부를 확인하고, 문법 오류, 참조 불일치, 타입 충돌 등 구조적 문제를 점검한다. 필요 시 추가 보정안을 제시하여 사용자가 최종 내용을 검토한 뒤 반영할 수 있도록 지원하며, 이를 통해 병합 이후 충돌을 수동으로 해결하는 부담을 줄이면서도 최종 통제권을 유지할 수 있다.

#### 세부 기능
- 작업 중인 워크트리 수집 및 변경 내용 비교
- 병합 전 충돌 예상 구간 식별
- AI 기반 병합 초안 및 중재안 제안
- 정적 분석 기반 문제 확인
- 필요 시 추가 보정안 제시
- 위험 구간 시각화 및 사용자 검토 지원
- 사용자 검토 후 최종 반영 지원

#### 기능적 의의
- 사후 충돌 해결이 아닌 사전 충돌 예방 가능
- 직접 충돌 및 일부 간접 충돌에 대한 선제적 대응
- 병합 안정성 향상
- 개발자의 수동 병합 부담 감소
- 자동화와 최종 통제권의 균형 확보

## 10. 병합 중재 범위
본 프로젝트는 병합 중재 범위를 단계적으로 설정하여 구현 가능성과 실효성을 동시에 확보하고자 한다.

### 10.1 1차 중재 범위
- 동일 파일 내 동일 코드 구간 변경
- 동일 함수 또는 인접 라인 수정
- 명시적 충돌 가능 구간 탐지

### 10.2 2차 중재 범위
- 공통 모듈 변경에 따른 영향 범위 식별
- 함수 시그니처 및 인터페이스 변경 감지
- 데이터 구조 변경에 따른 연결 지점 확인

### 10.3 3차 검증 범위
- 정적 분석 기반 문법 오류 점검
- 참조 불일치 및 타입 충돌 검토
- 병합 초안의 구조적 안정성 확인

## 11. 부가 기능

### 11.1 협업 텍스트 자동 추천
실제 변경 파일과 diff를 기반으로 커밋 메시지, 브랜치명, 작업 설명을 자동 추천한다. 사용자는 추천 결과를 그대로 적용하거나 수정 후 사용할 수 있다.

#### 세부 기능
- 커밋 메시지 추천
- 브랜치명 추천
- 작업 설명 추천
- 변경사항 요약 생성

### 11.2 로컬 브랜치 정리 지원
사용하지 않는 로컬 브랜치를 GUI 기반으로 선택 정리하거나, 일정 기준에 따라 자동 정리할 수 있도록 지원한다.

#### 세부 기능
- 로컬 브랜치 목록 조회
- GUI 기반 선택 정리
- 미사용 로컬 브랜치 자동 정리 지원

## 12. 시스템 동작 흐름
본 프로젝트에서 제안하는 시스템의 기본 동작 흐름은 다음과 같다.

1. 사용자가 AI 기반 코드 작업을 수행한다.
2. 시스템은 작업 수행 전 현재의 커밋되지 않은 로컬 작업 상태를 자동 스냅샷으로 저장한다.
3. 하네스 프롬프팅을 자동 적용하여 AI 작업의 일관성과 안전성을 높인다.
4. 작업 결과에 문제가 발생할 경우, 사용자는 해당 AI 작업 실행 직전 상태로 원복할 수 있다.
5. 병합 시점에는 시스템이 작업 중인 워크트리와 변경 내용을 수집한다.
6. 충돌 가능성이 높은 구간을 사전에 식별하고, AI 기반 병합 초안 및 중재안을 생성한다.
7. 정적 분석을 통해 제안된 병합 초안의 문제 여부를 점검하고, 필요 시 추가 보정안을 제시한다.
8. 사용자는 제안된 결과를 검토한 뒤 최종 반영 여부를 결정하고 병합을 진행한다.
9. 이후 필요 시 커밋 메시지, 브랜치명, 작업 설명 추천 기능과 로컬 브랜치 정리 기능을 활용한다.

## 13. 대표 사용자 시나리오
본 프로젝트의 대표 데모 시나리오는 다음과 같다.

1. 개발자는 커밋되지 않은 로컬 작업 상태가 존재하는 상황에서 AI에게 로그인 검증 로직 수정을 요청한다.
2. SafeGit은 AI 작업 실행 직전의 로컬 작업 상태를 자동 스냅샷으로 저장한다.
3. AI는 로그인 검증 로직과 관련 예외 처리 코드를 함께 수정한다.
4. 개발자는 결과를 검토하던 중 의도하지 않은 공통 유틸 변경이 함께 반영된 것을 확인한다.
5. 개발자는 해당 AI 작업 실행 직전 상태로 즉시 원복한다.
6. 이후 다른 브랜치와 병합하는 단계에서 SafeGit은 로그인 응답 형식 변경으로 인한 충돌 가능성을 탐지한다.
7. 시스템은 병합 초안과 중재안을 제시하고, 사용자는 제안 결과를 검토한다.
8. 사용자는 최종 반영 여부를 결정하고 병합을 진행한다.

상기 시나리오는 SafeGit의 핵심 가치인 ‘AI 작업 실행 단위 원복’과 ‘병합 전 충돌 중재’를 하나의 데모 흐름으로 확인할 수 있도록 구성한다.

## 14. 기술 구현 방안

### 14.1 자동 스냅샷 관리 방식
자동 스냅샷은 AI 작업 실행 직전의 커밋되지 않은 로컬 작업 상태를 기준으로 생성하며, 협업용 공식 이력과 분리된 내부 복구 레이어에서 관리한다. 이를 통해 사용자는 Git 히스토리를 오염시키지 않고도 작업 단위 복구 지점을 확보할 수 있다.

### 14.2 AI 작업 단위 식별 방식
AI 작업 단위는 개별 AI 요청 또는 실행 1회를 기준으로 식별한다. 각 AI 작업은 실행 직전 자동 스냅샷과 연결되며, 문제 발생 시 해당 작업 이전 상태를 기준으로 원복이 가능하도록 설계한다.

### 14.3 병합 충돌 분석 방식
병합 충돌 분석은 작업 중인 워크트리와 변경 내용을 수집한 뒤, 동일 파일, 동일 함수, 인터페이스 변경, 공통 모듈 영향 범위 등을 기준으로 충돌 가능성을 단계적으로 식별하는 방식으로 수행한다.

### 14.4 정적 분석 적용 방식
정적 분석은 AI가 제안한 병합 초안에 대해 문법 오류, 참조 불일치, 타입 충돌 등 구조적 문제를 사전에 점검하는 용도로 활용한다. 이를 통해 사용자는 병합 전 최소한의 안정성 검토 결과를 함께 확인할 수 있다.

### 14.5 사용자 반영 방식
AI는 병합 초안과 중재안을 제시하지만, 최종 반영 여부는 사용자가 직접 검토하고 결정하도록 설계한다. 이를 통해 자동화의 편의성과 사용자 통제 가능성을 동시에 확보한다.

본 프로젝트는 상기 기술 요소 중 자동 스냅샷 및 원복, 병합 전 충돌 탐지, 병합 초안 제안을 우선순위 1 MVP로 설정하여 핵심 가치가 직접적으로 검증될 수 있도록 구현 범위를 구성한다.

### 15.1 우선순위 1 MVP
- AI 작업 실행 직전 자동 스냅샷 생성

## 18. 차별성
기존 Git 도구는 공식 이력 관리와 충돌 발생 이후의 해결에 초점을 두는 반면, 본 프로젝트는 AI 작업 이전의 커밋되지 않은 로컬 상태를 자동으로 보존하고 작업 단위별 원복을 지원하는 세이프티 레이어를 제공한다는 점에서 차별성을 가진다. 즉, 협업용 기록인 커밋과 별개로, AI 작업에 특화된 경량 복구 체계를 제공한다는 점이 핵심적인 차별 요소이다.

또한 병합 이후 충돌을 해결하는 것이 아니라 병합 이전 단계에서 충돌 가능성을 예측하고 AI 기반 병합 초안 및 중재안을 제시한 뒤, 정적 분석을 통해 이를 검증하고 사용자가 최종 검토 후 반영할 수 있도록 지원한다는 점에서, 단순한 충돌 탐지 도구를 넘어 실제 병합 의사결정을 지원하는 확장 프로그램으로 발전 가능성을 가진다.
