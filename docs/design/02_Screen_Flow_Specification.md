# 02. 화면/플로우 명세서

### 1. 문서 목적

본 문서는 GitCat VS Code Extension의 전체 화면 구조와 사용자·시스템 플로우를 최종 명세한다.
기능 명세서, 시스템 아키텍처/모듈 설계서, 테스트/데모 시나리오 문서, 프로젝트 기획서를 종합하여 MVP 범위 내에서 구현 가능한 UI와 동작 흐름을 정의한다.

- **UI 구성 원칙**: VS Code Extension의 Sidebar Tree View와 Webview를 조합
- **범위**: MVP 우선순위 1·2 기능(자동 스냅샷·원복, 병합 전 충돌 예측·AI 중재, 추천 기능, 브랜치 정리) 전부 포함

### 3. 화면 상세 명세

### SCR-01 메인 GitCat View (Entry Point)

- 좌측 사이드바에 GitCat View 추가
- 3개의 대형 collapsible 섹션으로 구성
    - 1번 섹션: GIT 작업 및 AI
    - 2번 섹션: 안전 계층 (스냅샷)
    - 3번 섹션: 브랜치 정리
- 현재 브랜치, 변경 파일 수, 빠른 액션 버튼 제공

### SCR-02 GIT 작업 및 AI

- 하위 메뉴: 커밋, 브랜치, 병합, 푸시
- **커밋** 또는 **브랜치** 클릭 시 오른쪽 Side Panel에 프롬프트 입력 창 + AI 추천 결과 표시
- 추천 종류: 브랜치명(F-001), 커밋명(F-002), 커밋 메세지(F-003), PR description(F-004)
- 추천 결과는 즉시 적용·수정·복사 가능

### SCR-03 안전 계층 (스냅샷)

- 자동 생성된 스냅샷 목록 표시
- 각 항목에 스냅샷 제목, 파일 개수, AI 편집 태그, 생성 시간 표시
- “모든 기록 보기” 버튼 제공
- 항목 클릭 시 SCR-04 Webview로 이동

### SCR-04 스냅샷 상세 / 원복 확인 (Webview)

- 좌측: 스냅샷 정보
- 우측: 현재 상태와의 diff 비교
- “원복 실행” 버튼 (원복 전 안전 스냅샷 자동 생성 알림)

### SCR-05 충돌 후보 분석 결과

- 위험도별 Tree View (High/Medium/Low)
- 파일·함수·라인·간접 충돌 이유 표시
- “AI 병합 초안 생성” 버튼 (선택 후 SCR-06 이동)

### SCR-06 AI 병합 초안 검토 (Webview)

- 3-column 레이아웃
    1. 내 변경안
    2. 상대 변경안
    3. AI 병합 초안
- 중재안 설명, 정적 분석 결과(오류·경고·안전도)
- 최종 결정 버튼: **반영**, **수정 후 반영**, **반영하지 않음**

### SCR-07 브랜치 정리

- 체크박스 형태의 로컬 브랜치 목록
- 각 브랜치에 상태 태그(병합됨 / 오래됨 / 활성)
- “모두 선택” 체크박스
- 하단 AI 추천 영역
- “일괄 삭제” 버튼
- 자동 삭제 On/Off 토글

### 4. 주요 플로우 명세

### 플로우 1. AI 작업 보호 (Safety Layer)

1. AI 작업 실행 요청
2. Git 상태 수집 → 자동 스냅샷 생성 (SCR-03에 즉시 표시)
3. AI 작업 수행
4. 문제 발생 시 SCR-03 → 스냅샷 선택 → SCR-04
5. diff 확인 후 원복 실행 (원복 전 안전 스냅샷 자동 생성)

### 플로우 2. GIT 작업 → AI 추천

1. SCR-02에서 커밋 또는 브랜치 선택
2. 오른쪽 Side Panel에 프롬프트 입력
3. AI 추천 결과 표시 → 적용/수정

### 플로우 3. 병합 전 충돌 예측·AI 중재

1. SCR-02에서 병합 또는 충돌 분석 실행
2. SCR-05 충돌 후보 표시
3. 후보 선택 → AI 병합 초안 생성
4. SCR-06에서 3-way 비교·중재안·정적 분석 확인
5. 사용자 최종 결정 후 반영

### 플로우 4. 브랜치 정리

1. SCR-07 브랜치 정리 섹션 열기
2. 체크박스 선택 또는 모두 선택
3. AI 추천 확인
4. 일괄 삭제 또는 자동 삭제 On/Off 설정 후 실행

### 5. 화면 전환 다이어그램

메인 GitCat View (SCR-01)
├── GIT 작업 및 AI (SCR-02)
│     ├── 커밋/브랜치 → 오른쪽 AI 추천 패널
│     └── 병합 → 충돌 후보 (SCR-05) → AI 검토 (SCR-06)
├── 안전 계층 (스냅샷) (SCR-03) → 클릭 → 원복 (SCR-04)
└── 브랜치 정리 (SCR-07)

### 6. 사용자 경험 포인트

- AI 작업 직전 자동 스냅샷 실시간 생성
- 모든 AI 제안은 사용자 검토 후 최종 결정
- 3개 섹션 구조로 한눈에 GitCat 기능 접근 가능
- 오른쪽 프롬프트 패널로 자연스러운 AI 추천 경험
- 체크박스·On/Off·AI 추천 영역으로 브랜치 정리 편의성 극대화
