# 프로젝트 모노레포 (주제 미정)

이 저장소는 주제 미정 상태의 팀 프로젝트를 위한 **모노레포(Monorepo) 초기 골격**입니다.
Frontend, Backend, AI, Infra 구성을 하나의 저장소에서 관리하며, 이후 GitLab으로 자동 미러링될 예정입니다.

## 📂 폴더 구조 및 역할

- `apps/frontend/`: 프론트엔드 애플리케이션 (프론트엔드 개발자 작업 위치)
- `apps/backend/`: 백엔드 애플리케이션 (백엔드 개발자 작업 위치)
- `apps/ai/`: AI 모델 및 API 서버 (AI 개발자 작업 위치)
- `infra/`: 배포, 인프라 구성, 스크립트 등 (인프라 담당자 작업 위치)
- `docs/`: 기획, 아키텍처, 회의록 등 프로젝트 공통 문서 (팀 공용 및 문서 담당)
- `.github/workflows/`: GitHub Actions 등 CI/CD 파이프라인

## 🚀 작업 가이드

현재 `apps` 내부의 각 파트와 `infra` 폴더는 의도적으로 비워져 있습니다. 
각 파트 담당자가 자신이 맡은 디렉터리 내부 구조(src, package manager 설정 파일 등)를 직접 구성해야 합니다.

*   **프론트엔드 개발자**: `apps/frontend/` 내부에 초기 세팅을 진행해주세요.
*   **백엔드 개발자**: `apps/backend/` 내부에 초기 세팅을 진행해주세요.
*   **AI 개발자**: `apps/ai/` 내부에 초기 세팅을 진행해주세요.
*   **인프라 담당자**: `infra/` 내부 폴더 구조 설계 및 루트의 `docker-compose.yml`을 기반으로 확장해주세요.
*   **기획/문서 담당자**: `docs/` 하위 폴더에 관련 문서를 적재해주세요.

---
**향후 확장 방향**
각 서비스 초기 세팅 이후 `docker-compose.yml`에 컨테이너 설정들을 엮어 로컬 통합 테스트 환경을 구축하고, 완성도를 높여갈 예정입니다.
