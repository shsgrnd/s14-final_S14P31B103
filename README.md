# GitCat (안전한 작업 관리 및 스마트 병합 솔루션)

생성형 AI 코딩 환경에서의 안전한 작업 관리 및 스마트 병합을 지원하는 **Git 확장 프로그램** 프로젝트를 위한 모노레포(Monorepo)입니다.

## 📂 폴더 구조 및 역할

- `apps/extension/`: GitCat 클라이언트 애플리케이션 (VS Code 확장 프로그램 / 데스크톱 앱)
- `apps/backend/`: 사용자 메타데이터 관리, 변경 이력 및 정적 분석 서버 (백엔드 파트)
- `apps/ai/`: 하네스 프롬프팅 적용, 병합 초안/중재안 생성을 위한 AI 모델 API 서버 (AI 파트)
- `packages/`: 애플리케이션 간 공유되는 공통 비즈니스 로직 및 모듈
  - `git-manager/`: 로컬 Git 스냅샷 및 복구 관리 등 코어 Git 조작 모듈
  - `conflict-analyzer/`: 정적 분석 및 병합 전 충돌 가능성 분석 모듈
  - `prompt-utils/`: 병합 컨텍스트 파악 및 프롬프트 템플릿 관련 모듈
  - `shared-types/`: 클라이언트 / 백엔드 / AI 서버 간 공유 타입 및 스키마
- `infra/`: 배포, 인프라 구성, 스크립트 등
- `docs/`: 기획서(PRD), 아키텍처, 회의록 등 문서 내역

## 🚀 작업 가이드

각 모놀리식 모듈들은 분리되어 있으며, 파트별 담당자가 `apps/` 및 `packages/` 내부 환경을 설정합니다.

*   **클라이언트 담당자**: `apps/extension/` 내부에 초기 세팅을 진행해주세요.
*   **백엔드 담당자**: `apps/backend/` 내부에 초기 세팅을 진행해주세요.
*   **AI 개발자**: `apps/ai/` 내부에 초기 세팅 및 모델 연결을 진행해주세요.
*   **기획/문서 담당자**: `docs/` 하위 폴더에 관련 기획서를 적재 및 업데이트 해주세요.

---
**향후 확장 방향**
각 서비스 초기 세팅 이후 모노레포 워크스페이스(Yarn Workspace / pnpm 등)를 구성하고, `docker-compose.yml`을 통해 로컬 개발 환경(Backend & AI)을 구동하도록 개선합니다.
